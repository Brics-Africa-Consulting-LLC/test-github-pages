What is JSDoc?

JSDoc is a markup language used to annotate JavaScript source code files. It is intended to provide an easy way to create informative documentation from inline comments in the code. JSDoc generates documentation in HTML format from the annotations, allowing developers to quickly understand how different parts of the code work together.

Basic Usage of JSDoc

JSDoc uses annotations to describe the different parts of the code. These annotations start with a double forward slash // and are placed just above the part of the code they are describing.

@description The @description tag is used to provide a general description of a part of the code. This description is included in the generated documentation.

/**
 * @description Calculates the factorial of a number.
 */
function factorial(n) {
 if (n === 0) {
    return 1;
 }
 return n * factorial(n - 1);
}

@param : The @param tag is used to describe the parameters of a function. This includes the parameter's name, its type, and a description of its purpose.

/**
 * @description Calculates the sum of two numbers.
 * @param {number} a - The first number.
 * @param {number} b - The second number.
 * @returns {number} The sum of a and b.
 */
function add(a, b) {
 return a + b;
}

@returns : The @returns tag is used to describe the value that a function returns. This includes the return value's type and a description of its purpose.

/**
 * @description Converts a Celsius temperature to Fahrenheit.
 * @param {number} celsius - The temperature in Celsius.
 * @returns {number} The temperature in Fahrenheit.
 */
function convertToFahrenheit(celsius) {
 return celsius * 9 / 5 + 32;
}

To generate the documentation from your JSDoc annotations, you can use the jsdoc command in your terminal, specifying the input directory and the output directory. For example:

$ jsdoc -r ./src -d ./docs

This command will read all JavaScript files in the ./src directory and its subdirectories, parse the JSDoc annotations, and generate the documentation in HTML format in the ./docs directory. In our case, we used the configuration file, which just a json file with some set of instructions

{
  "source": {
    "include": ["./src"],
    "exclude": ["./node_modules"]
  },
  "opts": {
    "destination": "./docs"
  }
}

This is simply defining the source of files that should be considered and those that shouldn't. and the option for the destination for the generated documentation to be and the file that contains the main brief of the project (README.md). The generated documentation is saved in the docs folder

Additional Tags and Features JSDoc offers a wide range of tags and features to provide more detailed documentation for your code. Some of these tags include @module, @namespace, @type, @typedef, @class, @constructor, @property, @event, and many more. You can find a complete list of available tags and their usage in the official JSDoc documentation.
